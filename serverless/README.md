# Приём заявок → почта (Яндекс.Облако + Яндекс 360)

Заявки с формы принимает функция в **Яндекс.Облаке** и шлёт письмо на
**ediniyfpc@efpts.ru** через SMTP Яндекс 360. Всё в РФ, 152-ФЗ соблюдён.
Код — [`handler.py`](handler.py), зависимостей нет (только стандартная библиотека).

Схема: браузер → `fetch(ENDPOINT)` → функция Яндекс.Облака → SMTP Яндекс 360 → почта.

---

## Шаг 1. Пароль приложения в Яндекс 360 (2 мин)

Обычный пароль от почты для SMTP не подойдёт — нужен **пароль приложения**.

1. Войти в аккаунт ящика-отправителя (например `ediniyfpc@efpts.ru`) → [id.yandex.ru](https://id.yandex.ru) → **Безопасность** → **Пароли приложений**.
2. Создать пароль для типа **«Почта (SMTP)»**.
3. Скопировать выданный пароль — он понадобится на шаге 3 (вводится в облаке, мне присылать НЕ нужно).

> Можно завести отдельный ящик-отправитель, напр. `site@efpts.ru`, и слать с него на `ediniyfpc@efpts.ru`. Но проще слать с самого `ediniyfpc@efpts.ru` на него же.

## Шаг 2. Создать функцию

1. [console.yandex.cloud](https://console.yandex.cloud) → **Cloud Functions** → **Создать функцию**.
2. Среда выполнения — **Python 3.12**.
3. Способ — «Редактор кода»: создать файл `handler.py`, вставить содержимое [`handler.py`](handler.py).
   Точка входа — **`handler.handler`**.
4. Таймаут — **15 сек**, память — 128 МБ.

## Шаг 3. Переменные окружения функции

| Переменная | Значение |
|---|---|
| `SMTP_USER` | адрес ящика-отправителя, напр. `ediniyfpc@efpts.ru` |
| `SMTP_PASSWORD` | пароль приложения из шага 1 |
| `MAIL_TO` | `ediniyfpc@efpts.ru` (куда падают заявки) |
| `ALLOWED_ORIGIN` | `*` (или позже `https://ефпц.рф`) |

> Надёжнее хранить `SMTP_PASSWORD` в **Yandex Lockbox** и подключить как секрет, а не открытой переменной.

## Шаг 4. Публичный доступ

В настройках функции включить **«Публичная функция»** — чтобы её можно было вызвать из браузера.

## Шаг 5. Подключить к сайту

1. Скопировать **URL для вызова** функции — вид `https://functions.yandexcloud.net/<function-id>`.
2. Прислать этот URL мне (он не секретный) — я впишу его в `js/main.js` (константа `ENDPOINT`), запушу и протестирую.
   Либо самому: в `js/main.js` заменить `REPLACE_WITH_FUNCTION_ID` на id функции.

## Проверка

Отправить тестовую заявку с сайта → должно прийти письмо «Новая заявка с сайта Единый ФПЦ» на `ediniyfpc@efpts.ru` (проверить и «Спам»). Телефон лида — в теле письма, e-mail лида — в `Reply-To` (можно отвечать прямо из почты).
